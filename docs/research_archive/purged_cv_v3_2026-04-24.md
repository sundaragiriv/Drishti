# Flow Predictor v3 — Honest AUC via Purged K-Fold CV

**Date:** 2026-04-24
**Reference:** López de Prado, *Advances in Financial Machine Learning*, Ch. 7
**Code:** `signal_scanner/intelligence/purged_cv.py` + `research/eval_v3_purged.py`
**Review item:** #2

## Methodology

Naive temporal CV (the original v3 split — train ≤2023, val 2024, test 2025)
overstates AUC because rows on consecutive trading days share future-return
windows. A row at `t` with a 5-day forward label has its outcome partially
determined by data from `t+1..t+5`, which can also appear in adjacent
training rows.

**Purged K-Fold** fixes this:
1. Split data into K = 5 contiguous date folds.
2. For each fold k, train on all OTHER folds.
3. **Purge** training rows whose label window `[t, t+horizon]` overlaps with
   fold k's date range.
4. **Embargo** training rows that fall within `embargo_days` *after* fold k
   (slow indicators / rolling features may leak fold-k data forward).

Embargo = label horizon (3d for `label_3pct_3d`, 5d for `label_rr_2to1`).

## Results

### Naive temporal split (the original v3)

```
train=1,045,550  val=478,934  test=210,264
[label_3pct_3d]  val AUC = 0.6473   test AUC = 0.6518
[label_rr_2to1]  val AUC = 0.5981   test AUC = 0.5972
```

### Purged 5-fold CV

```
[label_3pct_3d]  horizon=3d  embargo=3d
  fold 1: AUC=0.6361   fold 2: AUC=0.6103   fold 3: AUC=0.6209
  fold 4: AUC=0.6262   fold 5: AUC=0.6369
  mean AUC = 0.6261  +/-0.0100

[label_rr_2to1]  horizon=5d  embargo=5d
  fold 1: AUC=0.5721   fold 2: AUC=0.5732   fold 3: AUC=0.5785
  fold 4: AUC=0.5878   fold 5: AUC=0.5807
  mean AUC = 0.5785  +/-0.0057
```

### Honesty delta

| Label | Naive val | Naive test | Purged mean | Δ vs val | Δ vs test |
|---|---|---|---|---|---|
| `label_3pct_3d` | 0.6473 | 0.6518 | **0.6261** ±0.010 | −0.021 | −0.026 |
| `label_rr_2to1` | 0.5981 | 0.5972 | **0.5785** ±0.006 | −0.020 | −0.019 |

## Interpretation

1. **The leakage was real, but bounded.** ~2 AUC points of the previously
   reported v3 performance was attributable to overlapping label windows.
   Both models still beat 0.5 by a meaningful margin.

2. **`label_3pct_3d` (Model A) has a real edge.** Honest AUC 0.626 with
   0.010 std across folds is statistically robust. This is roughly
   "moderate predictive power" in the López de Prado framework.

3. **`label_rr_2to1` (Model B) is borderline.** Honest AUC 0.578 with 0.006
   std means signal is real but small. After transaction costs (review #3 —
   ~0.15 R per trade at 1R-ATR stop frame), a 57.85% AUC may not produce
   positive expectancy at scale.

4. **Recommendation:** treat the honest AUCs as the ground truth for any
   future v4 / production gating decisions. Update memory + spec docs to
   reflect 0.626 / 0.578, not 0.65 / 0.60.

## Caveats

- Hyperparameters were tuned on naive validation. They may be slightly
  overfit to the leaky frame; properly nested purged CV with hyperparameter
  search would give a slightly lower number still. Skipped here for runtime.
- Purged CV with 1.7M rows × 5 folds × 2 labels took ~45 minutes on this
  machine (LightGBM with `n_estimators=2500`, `n_jobs=-1`).
- Composite-feature leakage (`ml_score_v2` and other intel scores recomputed
  each EOD — see review item #5) is NOT yet addressed by purged CV alone.
  That requires per-quarter historical scoring with the `ml_score_v2_version`
  + `ml_score_v2_scored_at` provenance columns added 2026-04-24. Tracking
  separately.
