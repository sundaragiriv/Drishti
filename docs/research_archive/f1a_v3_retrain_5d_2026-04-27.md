## What was tested

After F1's `label_hit_1R_10d` returned 0.513 AUC (no edge), F1-A tries the same
target at a shorter horizon — the actual production frame:

```
label_hit_1R_5d = 1 IFF
    high reaches close + 2*ATR20  BEFORE
    low touches  close - 2*ATR20  WITHIN
    next 5 trading days
```

This matches `swing_max_hold_days = 5` — the real maximum hold for the live
trader. The 10-day version was measuring outcomes the live system can't reach.

## Practical interpretation

At AUC 0.551 with base rate 21.2%, expected lift by decile:

| Decile (by predicted prob) | Expected hit rate |
|---|---|
| Top 10% | ≈ 31-33% |
| Top 20% | ≈ 27-29% |
| Middle 60% | ≈ 19-22% |
| Bottom 10% | ≈ 14-15% |

Top decile is ~50% above baseline — meaningful for ranking, not enough for
binary gating.

## Calibration finding (F2)

Isotonic regression fitted on val (2024). Result:

```
Val: 476,353 predictions ALL fell in bucket 0.20-0.30
  raw mean = 0.225  actual = 0.224  (miscalibration < 0.001)
Test: 199,728 predictions
  raw mean = 0.225  actual = 0.177  (~+0.05 overestimate)
```

The model's probabilities are tightly clustered around the base rate. It has
**rank-skill** (AUC 0.55) but **not magnitude-skill** (probabilities don't
spread out). Calibration on val transfers ~50% to test (regime shift).

**Implication:** use as a tie-breaker / ranking signal, not a confidence
gauge. "73% probability" doesn't mean what it would on a well-calibrated model.

## Decision: defer live integration

Wiring v3_5d into IdeaBridge as a ranking signal requires a runtime feature
pipeline that recomputes 47 base features + 12 interactions for every
candidate ticker. Mirror of `_build_features_v2.py` over live state.
Estimated effort: 1-2 days.

For Monday open we ship without it. The model artifact + calibrator are
preserved for the dedicated integration session.

In the meantime, our existing rule-based filters (Triple Lock at conv≥70 +
ml_v2≥70 + F4 insiders≥1, accumulation phase) are the production path.
