# Predictive AI Implementation Safeguards

Three mandatory safeguards for the Forward Return Predictor (fwd_v1).
These must be enforced before any training run produces a candidate model.

## 1. Point-in-Time Feature Integrity

Every feature must have a "known as of" date rule. No feature may use
information that was not publicly observable on the prediction date.

### Feature Family Rules

| Feature Family | Known-As-Of Rule | Implementation |
|----------------|------------------|----------------|
| Daily price/tech (SMA, RSI, ATR, returns) | Close of prediction date | Use `trade_date <= prediction_date` |
| Intelligence scores (conviction, phase, cascade) | Quarter filing deadline + 45 days | Map to prior settled quarter only. E.g., on 2024-03-15, use 2023-Q3 (filed by ~Feb 14), NOT 2023-Q4 (deadline May 15) |
| Form 4 insider transactions | Filing date (SEC EDGAR) | Use `filing_date <= prediction_date`, not transaction date |
| Short volume / dark pool | T+1 publication | Use `trade_date < prediction_date` (published next day) |
| Short interest | Settlement date + ~10 days | Use `settlement_date + 10 <= prediction_date` |
| Squeeze / pressure scores | Computed from above | Inherits strictest constituent rule |
| HMM regime state | Close of prediction date | Use regime predicted from bars up to prediction date |
| Sector rotation | Same as intelligence scores | Quarter-lagged |

### Enforcement

The feature join engine must accept a `as_of_date` parameter and filter
all source tables accordingly. Unit tests must verify that no feature row
contains data from after `as_of_date`.

## 2. Existing Model Outputs as Features

ML v2, Swing ML, and Strategy ML outputs are leakage-sensitive.

### Rules for v1

- **Exclude all existing model outputs from v1 feature set.**
  Reason: ML v2 was trained on 2020-2022 data but scored on 2023+.
  If the predictive model trains on 2021-2023 and uses ML v2 scores as
  features, those scores were generated by a model that saw some of the
  same training period. The leakage is indirect but real.

- **For v2+**, existing model outputs may be included only if:
  1. The source model was trained on data strictly before the predictive
     model's training start date
  2. Scores are frozen point-in-time snapshots (not recomputed)
  3. A leakage test confirms: removing source model features does not
     degrade OOS performance by more than the feature's apparent contribution

## 3. Quantile Coverage Validation

Platt scaling calibrates P(direction correct). The return-range bands
(25th/75th percentile from quantile regression) need separate validation.

### Required Checks

| Check | Threshold | How |
|-------|-----------|-----|
| 25th percentile coverage | 22-28% of actuals fall below | Empirical coverage on val set |
| 75th percentile coverage | 72-78% of actuals fall below | Empirical coverage on val set |
| Band width sanity | Median band width 2-8% for 5d | If narrower, model is overconfident; wider, uninformative |
| Coverage by regime | Passes in at least 4/5 HMM states | Regime-stratified coverage check |

### UI Rule

The ISR forecast card may only show the range band if:
- Empirical coverage is within 5pp of nominal (e.g., 25th pctl covers 20-30%)
- Otherwise, show only point estimate + direction probability

```
# Good calibration — show full card:
  Expected:  +2.4%
  Range:     +0.5% to +4.1%  (50% CI)

# Poor calibration — show reduced card:
  Expected:  +2.4%
  Direction: LONG (72% conf)
  Range:     not calibrated — omitted
```

## Pre-Training Checklist

Before any training run:

- [ ] Feature join engine has `as_of_date` parameter
- [ ] Intelligence features use settled quarter only
- [ ] Short data uses T+1 lag
- [ ] Form 4 uses filing_date not transaction_date
- [ ] No existing model outputs in feature set (v1)
- [ ] Unit test: no feature has data from after as_of_date
- [ ] Label pipeline: forward returns computed from close prices only
- [ ] Train/val/test split is strictly temporal (no shuffling)

## Post-Training Checklist

Before deployment:

- [ ] Direction accuracy > 55% on val set
- [ ] ECE < 0.05 (Platt-calibrated direction probability)
- [ ] IC > 0.05 (Spearman correlation, predicted vs actual)
- [ ] Quantile coverage within 5pp of nominal
- [ ] Sharpe > 1.5 (long-only, top decile, 5d hold)
- [ ] Profitable in at least 3/5 HMM regime states
- [ ] IC stable over 6-month rolling windows (no decay)
- [ ] No >80% same-direction predictions (discrimination check)
