**CHANGES REQUESTED**

**Summary**  
This module wraps the Codex CLI to run automated reviews either from git state or by embedding file contents into a prompt, then stores the review under `data/reviews`. The overall shape is workable, but the current implementation has execution-path bugs that can cause the MCP tool to report a successful review when Codex either failed to run or reviewed an incomplete prompt.

**Issues**  
- **Critical**: Git review mode passes an unsupported flag to `codex review`, so the command will fail immediately. [`mcp_server/codex_review.py`](/E:/Quant-Bridge/mcp_server/codex_review.py#L87) adds `--skip-git-repo-check`, but `codex review --help` shows that flag does not exist for `review` mode. As written, `--uncommitted` reviews in a real repo will error out.
- **Critical**: Command failures are still returned as `"status": "completed"`. In both execution branches, the code accepts `stdout`/`stderr`, then unconditionally returns completed without checking `result.returncode`. See [`mcp_server/codex_review.py`](/E:/Quant-Bridge/mcp_server/codex_review.py#L98), [`mcp_server/codex_review.py`](/E:/Quant-Bridge/mcp_server/codex_review.py#L138), and [`mcp_server/codex_review.py`](/E:/Quant-Bridge/mcp_server/codex_review.py#L171). That means MCP callers can receive a “successful” review that is actually just CLI error text.
- **Warning**: Non-git mode silently truncates the review prompt to 8000 characters and never uses the temp file it writes. [`mcp_server/codex_review.py`](/E:/Quant-Bridge/mcp_server/codex_review.py#L124) writes `prompt_file`, but [`mcp_server/codex_review.py`](/E:/Quant-Bridge/mcp_server/codex_review.py#L129) passes `review_prompt[:8000]` directly instead. Large diffs or multi-file reviews will be cut off, which defeats the purpose of automated review and can hide real findings.
- **Warning**: The `codex.exe` fallback is ineffective for PATH-installed Codex. [`mcp_server/codex_review.py`](/E:/Quant-Bridge/mcp_server/codex_review.py#L30) falls back to `Path("codex.exe")`, and [`mcp_server/codex_review.py`](/E:/Quant-Bridge/mcp_server/codex_review.py#L71) checks `.exists()`. If Codex is available via `PATH` rather than a file in the current directory, this will incorrectly return “not found” instead of attempting execution.

**Risk Assessment**  
**High**. The primary risk is false confidence: review requests can fail or run on incomplete input while the module reports success. That undermines the review automation path directly and makes downstream MCP consumers trust invalid results.
